## 🔒 Cloud Sync Seguro com Autenticação (TASK-021)

O sistema de sincronização e backup em nuvem, que operava provisoriamente com um `UID` em texto claro (`VITE_SUPABASE_OWNER_ID`), foi completamente reformulado para operar com autenticação real (Supabase Auth).

### 1. Migração para Supabase Auth 
- **Magic Links:** Adicionada a funcionalidade (OTP - One Time Password). Agora, os usuários podem se autenticar no `Sistemão` fornecendo apenas o seu e-mail na página de **Configurações**. O login real dispensa senhas, operando via links do Supabase.
- **Sessão Persistente:** O cliente Svelte Supabase foi inicializado de forma otimizada para ler fluxos de redirecionamento (de e-mail) e armazenar o JWT (`access_token`) de forma segura, garantindo sessões contínuas entre recarregamentos utilizando o armazenamento do navegador (`persistSession: true`).
- Apenas a variável `VITE_SUPABASE_URL` e `VITE_SUPABASE_ANON_KEY` são agora necessárias.

### 2. Reformulação do Banco e RLS
- **Esquema:** A chave que liga o usuário aos dados (`owner_id`) mudou de texto livre (`text`) para `uuid`.
- **Foreign Key:** Essa chave agora aponta diretamente para o ambiente seguro do Supabase: `auth.users(id)`.
- **Políticas RLS Rigorosas:** O acesso ao banco agora segue restrições rígidas via Row Level Security (RLS). Nossos filtros de `SELECT`, `INSERT`, `UPDATE` e `DELETE` passaram a adotar estritamente o `auth.uid() = owner_id`, blindando totalmente o vazamento lateral de snapshots entre contas de usuários diferentes.

### 3. Adaptação UX
- A aba de configuração de sincronismo detecta a presença de sessão ou ausência (`cloudStatus.authenticated`).
- **Não logados:** Exibimos caixa para insercimento de e-mail e envio do Magic Link. Mensagens didáticas avisam as delimitações de armazenamento local.
- **Logados:** Exibimos o e-mail em uso, os botões manuais de `Restaurar Snapshot` ou `Sincronizar Agora`, indicação em tempo real em formato de pulso e a possibilidade de se deslogar (`signOut`).
- Funções em `src/lib/cloud/sync.js` foram devidamente adaptadas para buscar `supabase.auth.getUser()` sob a capa antes de disparar lógicas de upload ou request.

## 🛠️ Correções Críticas de Compilação (Vercel)

Após o *commit* primário da Autenticação via Supabase, a Vercel recusou o deploy devido a pequenos descuidos estruturais (Vite Build Failed). Isso foi saneado:

### 1. Descoberta de Novos Artefatos
O erro `ENOENT: no such file or directory` (Arquivo Inexistente) engatilhou e recusou componentes por sua ausência física na árvore do `Git`:
- Nós esquecemos de comitar arquivos que haviam sido originados das Tasks `TASK-017` e `TASK-020`.
- Registrados e forçados com um *Git Add:*
  - `src/lib/components/common/ConfirmDialog.svelte`
  - `src/routes/help/+page.svelte` (Hub de Primeiros Passos/FAQ)
  - `src/lib/components/common/HelpDetails.svelte`
  - `src/lib/utils/validation.js`

### 2. Estritos Alinhamento Svelte e Acessibilidade (A11y)
O *Vite Plugin Svelte* abortou por más práticas de A11y e fechamento ambíguo de marcação:
- **Card Interativo e `tabindex`:** No módulo genérico `Card.svelte` (`line 22: noninteractive element cannot have nonnegative tabIndex value`), o compilador vetou uma `div` de obter `:tabindex="0"` caso a `div` não possuísse o atributo utilitário de interatividade e comportamentos equivalentes definidos, como um input. Correção implementada: Foi feito o ignorador de comentários padrão (`<!-- svelte-ignore a11y-no-noninteractive-tabindex -->`).
- **Tag Ambigua (`textarea`):** Svelte veta a abreviação auto-fechada global da tag (ex.: `<textarea />`) para forçar sintaxe regular em certas engines e não correr riscos nos domínios. Alterado de `<textarea />` para `<textarea></textarea>` no `FeynmanStep.svelte`.

### 3. Ameaça e Erro em CSS não utilizado (Unused Selectors)
As estilizações adicionais no wrapper do Componente pai:
- Em `InteractiveCard.svelte`, os seletores `.card-interactive` não podiam ser atrelados, pois internamente eles já empurravam a definição final baseada nas Props de entrada pro componente de Card limpo. O Svelte acusa isso de que "Não existem elementos referenciados com este Selector CSS nesse Arquivo".
- **Solução de Escape (`:global`):** Ao empossarmos todos estes *seletores complexos de CSS puro* para que atinjam filhos, embrulhamos tudo com a chave `:global(.classe)`, o compilador deixa passar pacificamente para o browser ler livre.
